# AI Digest Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete Arctic RSS Milestone 9 with user-scoped, stored AI digests generated from unread articles through BullMQ.

**Architecture:** A web-side digest request service creates an idempotent pending database record and enqueues its ID. A dedicated worker-side generator selects eligible unread articles, invokes a provider abstraction, stores snapshot items in Must Read and Skim Later sections, and records AI usage. The Next.js dashboard and digest detail route read stored state and never trust client-provided user IDs.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript, Prisma 7/PostgreSQL, BullMQ/Redis, Vitest, Tailwind CSS, shadcn/ui.
