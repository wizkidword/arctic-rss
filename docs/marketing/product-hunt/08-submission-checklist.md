# Submission checklist

## Before creating the draft

- [ ] `00-status.md` says GO, not NO-GO.
- [ ] Personal maker account is eligible and recorded in `human-inputs.md`.
- [ ] Pricing is confirmed.
- [ ] P1 guest-preview de-duplication fix is deployed and production-smoked.
- [ ] Final gallery and optional demo are captured only from approved public/demo data.
- [ ] `npm run ph:assets:validate` passes under Node 22+.
- [ ] Local preview page has been reviewed by a human.

## Manual Product Hunt draft

- [ ] Maker logs in personally and creates the draft manually.
- [ ] Name: `Arctic RSS`.
- [ ] Direct URL: `https://arcticrss.com`.
- [ ] Tagline is 60 characters or fewer.
- [ ] Use the 229-character description unless the live form allows the extended fallback.
- [ ] Select at most three relevant live tags.
- [ ] Upload reviewed thumbnail and ordered gallery assets.
- [ ] Add YouTube only after the video is public/playable, captioned, and has a full non-private URL.
- [ ] Confirm maker profiles and any tool shoutouts are truthful.
- [ ] Preview every field, image, link, and the live product in a logged-out browser.
- [ ] Write the first comment personally from `maker-comment-workbook.md`; rewrite every sentence in personal words.
- [ ] Schedule only when the maker can actively support the relevant launch window. 12:01 a.m. Pacific is optional, not a target by itself.

## Stop conditions

- Stop if the guest reader is empty, duplicate, broken, or misleading.
- Stop if the direct product URL fails or shows a different release.
- Stop if any asset includes private data, unsupported claims, or broken content.
- Stop if personal account eligibility, pricing, or maker identity is unknown.
- Stop if the maker cannot personally monitor and respond.
